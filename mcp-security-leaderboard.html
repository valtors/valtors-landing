mcp security leaderboard

you connected an MCP server to your AI agent. you didn't check what was inside.

nobody does. there are thousands of MCP servers on npm. each one exposes tools to your LLM. each tool has a description. each description is text your agent reads and obeys.

we scanned 17 of them. with mcprobe. real servers, real scores, real findings. no anonymizing. no "trust me bro." the output speaks.

some of these servers tell your agent to ignore your instructions. and your agent will.

methodology

how we scored them.

each server was started via npx -y <package> and scanned with mcprobe. mcprobe enumerates tools, parses descriptions, checks for injection patterns, unrestricted file paths, missing schemas, and tool name shadowing. every finding is real. every score is calculated. nothing is opinion.

risk scores: CRITICAL = prompt injection found (tool descriptions containing "ignore all previous instructions" or "you are now a"). HIGH = unrestricted file paths or sensitive resource exposure. MEDIUM = injection-adjacent patterns, missing descriptions, oversized descriptions. LOW = missing schemas, name shadowing, unrestricted parameters.

a score of 0 doesn't mean safe. it means we didn't find anything. there's a difference. you should know the difference.

disclaimer

a score of 0 doesn't mean safe.

it means we didn't find anything. there's a difference. you should know the difference.

mcprobe checks tool descriptions, schemas, and parameter patterns. it doesn't execute tool calls. it doesn't test runtime behavior. it doesn't check the server's network calls. a server can score 0 and still exfiltrate your data at runtime. a server can score 100 and be perfectly safe if you never connect it to an LLM that follows instructions.

but you will connect it to an LLM that follows instructions. that's the point of MCP. so maybe check what's inside first.
