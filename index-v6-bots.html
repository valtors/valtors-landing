What can Relay do?

Each agent has a role. Together, they ship.

🎯

Plan your sprint

PM agent breaks down features into tasks, assigns priorities, and creates GitHub issues automatically.

⚡

Write the code

Dev agent picks up tasks, writes implementation, handles tests, and opens pull requests for review.

🔍

Review and ship

QA agent reviews PRs, runs checks, catches bugs before merge, and confirms deployment health.

📝

Write the docs

Docs agent keeps README, changelogs, and API references up to date as code changes land.

🚀

Launch day

Marketing agent drafts tweets, writes launch posts, and preps Product Hunt materials from your roadmap.

The Problem

AI agents forget everything

Every new chat starts from zero. Your agent doesn't remember yesterday's decisions, last week's architecture, or why you chose Postgres over Mongo. Relay fixes this with persistent context that flows between agents like institutional memory.

Three commands. That's it.

1

Install Relay

One binary, zero dependencies. Works on Mac, Linux, Windows.

go install github.com/valtors/relay@latest
2

Connect your MCP client

Add Relay to Claude Desktop, Cursor, or any MCP-compatible tool. Takes 10 seconds.

relay serve --port 3000
3

Let your agents cook

Agents pass context, checkpoint progress, and pick up where others left off. You review and approve.

relay status
$ relay status

pm-agent ✓ 3 tasks created
dev-agent ✓ auth module shipped
qa-agent ● reviewing PR #14...
docs-agent ◌ waiting for merge

✓ 2 checkpoints saved
context: 12.4kb shared across agents
