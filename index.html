03 / the workbench

eight tools. one philosophy.

boring tech, zero dependencies where possible, built to outlast the hype cycle.

01
Relay
one binary. forty tools. zero dependencies.
MCP server with 40 tools for AI agents. files, images, PDFs, web fetch, workflows. stdio and HTTP transport. other people install five servers to do what this does alone. npm: userelay.
github ->
02
Reflow
responsive without breaking SSR. eight frameworks. one API.
SSR-safe responsive toolkit for TypeScript. breakpoints, container queries, fluid typography, viewport tracking. React, Vue, Svelte, Solid, Qwik, Preact, Angular. no hydration mismatch warnings. no framework lock-in. npm: usereflow.
github ->
03
Observer
your agent made thirty tool calls. you saw the text. this shows the rest.
transparent MCP proxy for agent observability. logs every tool call to SQLite, exposes trace history, reduces token overhead. it sits between your client and the server and watches. silently.
github ->
04
mcprobe
you connected a server you didn't audit. this tells you what you missed.
security scanner for MCP servers. finds prompt injection in tool descriptions, tool shadowing, resource exposure to sensitive paths, baseline drift. your MCP server is probably vulnerable. single Go binary.
github ->
05
Vault
run untrusted tools without burning down your machine.
strips secrets from environment, blocks sensitive file paths, scans for prompt injection in tool descriptions, logs every action to SQLite. not a real sandbox. an audit layer. honest about what it is.
github ->
06
Pulse
thirty-eight notifications. three urgent items. your AI does the rest.
connect everything. github, gmail, calendar in one place. asks "what did i miss" and gets a real answer. remembers what matters, forgets what doesn't. local-first, single binary. not a framework. a product.
github ->
07
Cairn
agents forget. this makes them remember.
temporal knowledge store in one sqlite file. facts are triples with validity windows. contradictions close old facts, they don't delete them. forgetting is a first-class operation. no neo4j. no cloud. no lock-in.
github ->
08
Smith
npm made javascript installable. this does it for MCP.
install, compose, secure, and manage MCP servers. one binary. fetch from npm, git, or local path. spawn all installed servers and expose one unified endpoint. your work tools don't leak into your personal profile.

04 / the principles

everything you need. nothing you don't.

01
MIT licensed. free to use, modify, and build on. no proprietary hooks, no vendor lock-in, no telemetry. if you find telemetry in here, it's a bug.
02
Local first. single binary, sqlite, go stdlib. your data doesn't leave your machine. boring tech on purpose because boring tech doesn't break at 3am.
03
MCP native. built for the model context protocol from day one. stdio transport, stdlib JSON-RPC. no magic, no abstraction layers, no frameworks on top of frameworks.
04
Security first. mcprobe finds prompt injection, tool shadowing, resource exposure. observer redacts sensitive data by default. security is built in. not bolted on after someone files an issue.
05
No corporate backing. no VC money. no roadmap dictated by a sales team. built with feedback from developers. contributions welcome. external contributors shipping.
